# refactor-engine: リファクタリング

既存コードを可読性・保守性・パフォーマンスの観点から改善する。

## 改善観点

| 観点 | チェック内容 |
|---|---|
| 可読性 | 関数名・変数名が意図を表しているか |
| 単一責任 | 1関数が1つのことをしているか |
| DRY | 重複コードが排除されているか |
| パフォーマンス | 不要なレンダリング・クエリがないか |

## プロセス

1. コードを読んで問題箇所を特定
2. 改善方針を提示（変更前に確認）
3. 段階的に変更（一度に全変更しない）
4. 変更後のテスト実行を確認
