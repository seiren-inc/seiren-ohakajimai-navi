# debugger: バグ特定・修正

エラーの根本原因を特定し修正する。

## デバッグプロセス

1. **症状の確認**: エラーメッセージ・再現手順を把握
2. **仮説立案**: 原因の候補を3つ列挙
3. **検証**: 最もシンプルな仮説から検証
4. **修正**: 根本原因に対処する（symptom fixしない）
5. **検証**: 修正後に同じエラーが出ないことを確認

## よくある原因

- import パスの誤り
- 非同期処理のawait漏れ
- null/undefined の未チェック
- 型の不一致
- 環境変数の未設定
