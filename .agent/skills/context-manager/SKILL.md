# context-manager: コンテキスト最適化

タスクに必要な情報のみをロードし、コンテキストウィンドウを節約する。

## 戦略

1. **遅延ロード**: 必要になった時点でSKILL.mdを読み込む
2. **圧縮**: 長いコンテキストを要点のみに圧縮する
3. **優先度付け**: タスク完了に直結する情報を最優先で保持する

## ロードルール

- 実行中のSkillのみロード（完了したものはアンロード）
- コードファイルは該当箇所のみ参照（全ファイル読み込み禁止）
- エラーメッセージは最新3件のみ保持
