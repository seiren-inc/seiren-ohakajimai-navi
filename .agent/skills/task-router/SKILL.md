# task-router: タスク振り分け

大きなタスクを実行可能な単位に分解して処理する。

## 分解ルール

1. タスクを **設計・実装・検証** の3フェーズに分割
2. 各フェーズを独立した実行単位に変換
3. 依存関係を特定して実行順序を確定

## 出力フォーマット

```markdown
## タスク分解
- [ ] フェーズ1: 設計 → [design-skill]
- [ ] フェーズ2: 実装 → [impl-skill]
- [ ] フェーズ3: 検証 → [qa-checklist]
```
